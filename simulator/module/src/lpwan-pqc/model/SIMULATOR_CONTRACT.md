# Updated simulator contract — lpwan-pqc

This artifact contains a cleaned, documented version of the ns-3 lpwan-pqc simulator.

## Scientific primitive

The simulator is exchange-driven.

A single call to:

`PqcExchangeApplication::RunOneExchange(nowS, logicalChannels)`

returns exactly one `ExchangeResult`, and that structure is the single source of truth for CSV export.

## Exchange semantics

One exchange models a fragmented PQC transfer composed of:
- PK fragments
- CT fragments
- ACK frames generated by the batching logic

The exported row represents the complete outcome of that exchange.

## Failure semantics

A data or ACK frame can fail because of:
1. PHY/channel decoding failure (`p_phy`)
2. collision failure (`p_col`)
3. deterministic receiver unavailability due to RX duty cycle

Probability composition is centralized at application level as:

`p_total = 1 - (1 - p_phy) * (1 - p_col)`

RX duty cycle is not probabilistic. Reception succeeds only when the full frame airtime interval is fully contained in an active RX window.

## Module responsibilities

- `PqcExchangeApplication`: exchange protocol, retransmissions, ACK batching, metrics
- `LpwanAlohaCollisionModel`: collision/interference semantics only
- `LpwanPacketErrorModel`: PHY/channel PER only
- `LpwanLoraAirtime`: airtime and fragmentation utilities only
- `PerTable`: PER(SNR) lookup/interpolation only
- `LpwanPqcHelper`: ns-3 object wiring only
- `scratch/lpwan-pqc-export-exchanges.cc`: reproducible experiment control and CSV export only

## Logging policy

Minimal scientific logging is preserved for:
- `INITIALIZE_EXCHANGE_RESULT`
- `ATTEMPT_BEGIN`
- `ATTEMPT_RESULT`
- `APPLY_FRAGMENT_GAP`
- `APPLY_DATA_ATTEMPT_GAP`
- `APPLY_ACK_RECOVERY_GAP`
- `FINALIZE_EXCHANGE_OUT`

Non-contractual compute-time logging was removed.

## Updated consistency fixes in this artifact

- `SfAwarePacingThreshold` default updated from `255` to `7` so radio-aware pacing is active by default for LoRa SF 7..12.
- `DoesIntervalOverlapRxWindow(...)` renamed to `IsFullyInsideRxWindow(...)` to match the true contract semantics.
- Scratch CSV export remains aligned 1:1 with `ExchangeResult`.
